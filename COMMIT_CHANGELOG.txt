FILE MODIFICATI/CREATI (in repo git)
------------------------------------

A. run-dynamic.py [NEW]
   Wrapper Python che genera working-hours dinamiche basate sull'ora di lancio.
   - Sempre 5 sessioni (configurabile con --sessions N)
   - Durata 90 min ciascuna, gap inizio-inizio 3h con jitter ±15 min
   - Vincolo anti-ban: solo tra 09:00 e 23:30
   - Auto-shrink del gap se le 5 sessioni non ci stanno entro le 23:30
   - Modalità --dry-run per anteprima senza lanciare
   Uso: python run-dynamic.py [--sessions N] [--duration-min M] [--gap-h H]

B. run-bot.sh [NEW]
   Wrapper bash per alternare follow/unfollow a giorni alterni.
   - Giorni dispari → config.yml (follow + like)
   - Giorni pari    → config-unfollow.yml (solo unfollow)
   - Modalità manuale: ./run-bot.sh follow | unfollow
   Cron suggerito: 0 9 * * * cd /path/bot && ./run-bot.sh

C. .idea/runConfigurations/ [NEW] (4 configurazioni PyCharm)
   - Bot - Lancio dinamico (5 sessioni)  → main run-dynamic.py
   - Bot - Anteprima sessioni (dry-run)  → mostra orari senza lanciare
   - Bot - Unfollow (config-unfollow)    → solo unfollow
   - Bot - Stop (kill processi)          → killa tutti i processi bot

D. GramAddict/core/views.py [MODIFIED]
   Modifiche minori per compatibilità UI 300.x (selettori fallback).

E. GramAddict/core/filter.py [MODIFIED]

PIPELINE DECISIONALE BOT (ordine di check per ogni profilo)
----------------------------------------------------------
1. Già interagito di recente?         → skip (storico)
2. Dado interact-percentage 60-75%    → skip casuale (umanizza)
3. Profilo non disponibile/vuoto      → skip
4. Tu lo segui già?                   → skip (skip_following)
5. Lui ti segue già?                  → skip (skip_follower)
6. È privato?                         → skip (skip_if_private)
7. min/max followers                  → skip se fuori range
8. min/max followings                 → skip se fuori range
9. potency_ratio (followers/foll)     → skip se < 0.25 o > 15
10. min_posts                         → skip se < 6
11. PASS → like 2 post + dado follow 50-65% → follow se favorevole
12. Anti-block check dopo ogni azione

STRATEGIA ATTUALE: "BURST & REST" (vs. "DISTRIBUTED")
-----------------------------------------------------
Il bot fa 4-5 sessioni di 20-30 min ciascuna, con ~2h di pausa tra una e l'altra.
Dentro la sessione il rate orario e' ALTO (~25-30 follow/h, ~50/h like) ma compensato
dalle lunghe pause -> rate medio giornaliero rispetta le tabelle.

PRO:
  - Pattern simile a "utente che apre IG 5 volte/giorno"
  - Apri/chiudi app meno volte = meno fingerprint
  - Sessioni piu' produttive (la sorgente non si "raffredda")

CONTRO:
  - Rate orario in-sessione fuori dalle tabelle "prudenti" (8-10/h)
  - Se IG usa rate-limiter su finestra <= 1h, possibile soft action-block

ALTERNATIVA "DISTRIBUTED" (da attivare SOLO se vediamo problemi):
  working-hours: 8-10 micro-sessioni di 20-25 min
  total-follows-limit: 6-8 per sessione
  -> rate sempre <= 8/h, nessun burst.

QUANDO PASSARE A DISTRIBUTED:
  - Comparsa di action-block ("Try again later")
  - Tasso follow-back < 5% dopo 7 giorni
  - Reset password forzato da IG
  - Verifiche identita' richieste

DECISIONE: si rimane su "BURST" per i primi 7 giorni di osservazione.
Risultato 02/05/2026 (giorno 1):  51 follow, 104 like, 0 errori, 0 action-block.
